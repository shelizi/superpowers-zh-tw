# 規格文件審查員提示模板

調度規格文件審查員子代理時使用此模板。

**用途：** 驗證規格是否完整、一致，並為實作計畫做好準備。

**調度時機：** 規格文件寫入 docs/superpowers/specs/ 之後

```
Task tool（通用）:
  description: "審查規格文件"
  prompt: |
    你是一名規格文件審查員。驗證此規格是否完整並準備好進行計畫編寫。

    **待審查規格：** [SPEC_FILE_PATH]

    ## 檢查內容

    | 類別 | 檢查要點 |
    |------|----------|
    | 完整性 | TODO、佔位符、"TBD"、不完整的章節 |
    | 一致性 | 內部矛盾、相互衝突的需求 |
    | 清晰度 | 需求模糊到可能導致建構出錯誤的東西 |
    | 範圍 | 是否足夠聚焦以用於單個計畫——而非涵蓋多個獨立子系統 |
    | YAGNI | 未請求的功能、過度設計 |

    ## 校準標準

    **只標記會在實作計畫階段造成實際問題的事項。**
    遺漏的章節、矛盾之處、或者模糊到可能被兩種不同方式理解的需求——
    這些才是問題。措辭上的小改進、風格偏好、以及「某些章節不如其他章節詳細」則不是。

    除非存在會導致計畫出錯的嚴重缺陷，否則應予以通過。

    ## 輸出格式

    ## 規格審查

    **狀態：** 通過 | 發現問題

    **問題（如有）：**
    - [章節 X]：[具體問題] - [為什麼這對計畫編寫很重要]

    **建議（僅供參考，不阻止通過）：**
    - [改進建議]
```

**審查員返回：** 狀態、問題（如有）、建議
