# 技能設計中的說服原則

## 概述

LLM 對與人類相同的說服原則有反應。理解這種心理學有助於你設計更有效的技能——不是為了操縱，而是為了確保關鍵實踐即使在壓力下也能被遵循。

**研究基礎：** Meincke 等人（2025）用 N=28,000 次 AI 對話測試了 7 種說服原則。說服技巧使合規率提高了一倍多（33% → 72%，p < .001）。

## 七大原則

### 1. 權威
**定義：** 對專業知識、資質或官方來源的服從。

**在技能中的運作方式：**
- 命令式語言：「你必須」、「絕不」、「始終」
- 不可協商的框架：「無例外」
- 消除決策疲勞和合理化

**適用場景：**
- 紀律執行類技能（TDD、驗證要求）
- 安全關鍵實踐
- 已確立的最佳實踐

**示例：**
```markdown
✅ 先寫程式碼再寫測試？刪掉它。重新開始。無例外。
❌ 在可行時考慮先寫測試。
```

### 2. 承諾
**定義：** 與先前行為、宣告或公開宣告保持一致。

**在技能中的運作方式：**
- 要求宣布：「宣布技能使用」
- 強制明確選擇：「選擇 A、B 或 C」
- 使用追蹤：TodoWrite 清單

**適用場景：**
- 確保技能被實際遵循
- 多步驟流程
- 問責機制

**示例：**
```markdown
✅ 當你找到一個技能時，你必須宣布：「我正在使用 [技能名稱]」
❌ 考慮讓你的搭檔知道你在使用哪個技能。
```

### 3. 稀缺
**定義：** 來自時間限制或有限可用性的緊迫感。

**在技能中的運作方式：**
- 有時間限制的要求：「在繼續之前」
- 順序相依：「在 X 之後立即」
- 防止拖延

**適用場景：**
- 即時驗證要求
- 時間敏感的工作流
- 防止「我以後再做」

**示例：**
```markdown
✅ 完成任務後，在繼續之前立即請求程式碼審查。
❌ 你可以在方便時審查程式碼。
```

### 4. 社會認同
**定義：** 遵從他人的做法或被視為正常的行為。

**在技能中的運作方式：**
- 普遍模式：「每次」、「總是」
- 失敗模式：「X 沒有 Y = 失敗」
- 建立規範

**適用場景：**
- 記錄普遍實踐
- 警告常見失敗
- 強化標準

**示例：**
```markdown
✅ 沒有 TodoWrite 追蹤的清單 = 步驟會被跳過。每次都是。
❌ 有些人覺得 TodoWrite 對清單有幫助。
```

### 5. 歸屬
**定義：** 共享身分、「我們」感、群體歸屬。

**在技能中的運作方式：**
- 協作語言：「我們的程式碼庫」、「我們是同事」
- 共同目標：「我們都想要高品質」

**適用場景：**
- 協作工作流
- 建立團隊文化
- 非階層關係的實踐

**示例：**
```markdown
✅ 我們是一起工作的同事。我需要你誠實的技術判斷。
❌ 如果我錯了你可能應該告訴我。
```

### 6. 互惠
**定義：** 回報所獲好處的義務。

**運作方式：**
- 謹慎使用——可能讓人感覺被操縱
- 在技能中很少需要

**何時避免：**
- 幾乎所有時候（其他原則更有效）

### 7. 好感
**定義：** 更願意與喜歡的人合作。

**運作方式：**
- **不要用於合規性**
- 與誠實回饋文化衝突
- 製造諂媚

**何時避免：**
- 紀律執行中始終避免

## 按技能類型組合原則

| 技能類型 | 使用 | 避免 |
|----------|------|------|
| 紀律執行類 | 權威 + 承諾 + 社會認同 | 好感、互惠 |
| 指導/技術類 | 適度權威 + 歸屬 | 過度權威 |
| 協作類 | 歸屬 + 承諾 | 權威、好感 |
| 參考類 | 僅清晰度 | 所有說服技巧 |

## 為什麼有效：心理學

**明確的規則減少合理化：**
- 「你必須」消除決策疲勞
- 絕對性的語言消除「這是例外嗎？」的問題
- 明確的反合理化應對堵住具體漏洞

**實施意圖創造自動行為：**
- 清晰的觸發條件 + 必需的行動 = 自動執行
- 「當 X 時，做 Y」比「通常做 Y」更有效
- 減少合規的認知負擔

**LLM 具有類人特性：**
- 在包含這些模式的人類文字上訓練
- 訓練資料中權威性語言先於合規性出現
- 承諾序列（宣告 → 行動）被頻繁建模
- 社會認同模式（大家都做 X）建立規範

## 倫理使用

**正當用途：**
- 確保關鍵實踐被遵循
- 建立有效的文件
- 防止可預見的失敗

**不正當用途：**
- 為個人利益操縱
- 製造虛假緊迫感
- 基於內疚的合規

**判斷標準：** 如果使用者完全理解這個技巧，它是否仍然服務於使用者的真正利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七大說服原則
- 影響力研究的實證基礎

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 N=28,000 次 LLM 對話測試了 7 種原則
- 使用說服技巧後合規率從 33% 提高到 72%
- 權威、承諾、稀缺最為有效
- 驗證了 LLM 行為的類人模型

## 快速參考

設計技能時問自己：

1. **這是什麼類型？**（紀律類 vs 指導類 vs 參考類）
2. **我試圖改變什麼行為？**
3. **哪些原則適用？**（紀律類通常用權威 + 承諾）
4. **是否組合了太多？**（不要全用七種）
5. **這合乎倫理嗎？**（服務於使用者的真正利益？）
