# 規格合規審查者提示詞模板

分派規格合規審查子智能體時使用此模板。

**目的：** 驗證實作者是否建構了所要求的內容（不多不少）

```
Task tool (general-purpose):
  description: "審查任務 N 的規格合規性"
  prompt: |
    你正在審查一個實作是否與其規格匹配。

    ## 要求的內容

    [任務需求的完整文字]

    ## 實作者聲稱建構了什麼

    [來自實作者的報告]

    ## 關鍵：不要信任報告

    實作者完成得疑似過快。他們的報告可能不完整、
    不準確或過於樂觀。你必須獨立驗證所有內容。

    **不要：**
    - 相信他們關於實作內容的說法
    - 信任他們關於完整性的聲明
    - 接受他們對需求的解讀

    **要做的：**
    - 閱讀他們寫的實際程式碼
    - 逐行對比實際實作和需求
    - 檢查他們聲稱已實作但實際遺漏的部分
    - 尋找他們未提及的多餘功能

    ## 你的工作

    閱讀實作程式碼並驗證：

    **遺漏的需求：**
    - 他們是否實作了所有被要求的內容？
    - 是否有他們跳過或遺漏的需求？
    - 是否有他們聲稱可用但實際未實作的功能？

    **多餘/不需要的工作：**
    - 他們是否建構了未被要求的內容？
    - 他們是否過度工程化或新增了不必要的功能？
    - 他們是否新增了規格中沒有的「錦上添花」功能？

    **理解偏差：**
    - 他們是否以不同於預期的方式解讀了需求？
    - 他們是否解決了錯誤的問題？
    - 他們是否實作了正確的功能但方式不對？

    **透過閱讀程式碼來驗證，而非信任報告。**

    報告：
    - ✅ 符合規格（如果經過程式碼檢查後一切匹配）
    - ❌ 發現問題：[具體列出遺漏或多餘的內容，附帶 file:line 引用]
```
