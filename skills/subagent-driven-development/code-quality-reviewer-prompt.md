# 程式碼品質審查者提示詞模板

分派程式碼品質審查子智能體時使用此模板。

**目的：** 驗證實作是否建構良好（整潔、有測試、可維護）

**僅在規格合規性審查通過後才分派。**

```
Task tool (superpowers:code-reviewer):
  使用模板 requesting-code-review/code-reviewer.md

  WHAT_WAS_IMPLEMENTED: [來自實作者的報告]
  PLAN_OR_REQUIREMENTS: [plan-file] 中的任務 N
  BASE_SHA: [任務開始前的提交]
  HEAD_SHA: [當前提交]
  DESCRIPTION: [任務摘要]
```

**除標準程式碼品質關注點外，審查者還應檢查：**
- 每個檔案是否有單一明確的職責和定義清晰的介面？
- 各單元是否拆分得足以獨立理解和測試？
- 實作是否遵循了計畫中的檔案結構？
- 本次實作是否建立了已經很大的新檔案，或顯著增大了現有檔案？（不要標記已有的檔案大小問題——聚焦於本次變更帶來的影響。）

**程式碼審查者返回：** 優點、問題（關鍵/重要/次要）、評估結論
