# 計畫文件審查員提示範本

調度計畫文件審查員子代理時使用此範本。

**用途：** 驗證計畫是否完整、與規格匹配，並且任務分解合理。

**調度時機：** 完整計畫編寫完成後。

```
Task tool（通用）:
  description: "審查計畫文件"
  prompt: |
    你是一名計畫文件審查員。驗證此計畫是否完整並準備好進行實作。

    **待審查計畫：** [PLAN_FILE_PATH]
    **參考規格：** [SPEC_FILE_PATH]

    ## 檢查內容

    | 類別 | 檢查要點 |
    |------|----------|
    | 完整性 | TODO、佔位符、不完整的任務、遺漏的步驟 |
    | 規格對齊 | 計畫覆蓋了規格需求，沒有重大範圍蔓延 |
    | 任務分解 | 任務有清晰的邊界，步驟可執行 |
    | 可建構性 | 工程師能否按此計畫執行而不會卡住？ |

    ## 校準標準

    **只標記會在實作階段造成實際問題的事項。**
    實作者建構了錯誤的東西或卡住了——這是問題。
    措辭上的小改進、風格偏好和「錦上添花」的建議則不是。

    除非存在嚴重缺陷——規格中的需求遺漏、
    矛盾的步驟、佔位內容、或者模糊到無法執行的任務——否則應予以通過。

    ## 輸出格式

    ## 計畫審查

    **狀態：** 通過 | 發現問題

    **問題（如有）：**
    - [任務 X，步驟 Y]：[具體問題] - [為什麼這對實作很重要]

    **建議（僅供參考，不阻止通過）：**
    - [改進建議]
```

**審查員返回：** 狀態、問題（如有）、建議
